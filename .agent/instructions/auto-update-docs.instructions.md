# .github docs auto-update instruction
# 10-word summary: Always sync .github markdown docs with project after code changes

## Rule
After any code, file, or directory change in the workspace, always update all .md documentation files in the .github folder to accurately reflect the current state of the project. This includes changes from adding, removing, or modifying code, files, or directories.

## Scope
- Applies to all .md files in the .github folder
- Applies to all code, file, and directory changes in the workspace
- Enforced for all contributors and automation

## Rationale
Keeping documentation synchronized with the codebase ensures accuracy, reduces confusion, and improves maintainability. Outdated docs can lead to errors and slow onboarding.

## Example Prompts
- "Update .github docs after refactoring the codebase."
- "Sync .github markdown files with the latest project structure."
- "Auto-update documentation after adding or removing files."
